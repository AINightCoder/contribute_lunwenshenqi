# 设计牵头EPC 项目全过程风险管理及应对措施探讨

王闯

（中国建筑第二工程局有限公司核电建设分公司，广东深圳518000)

摘要：目前，EPC工程总承包项目已在建设工程施工中得到了广泛应用，这种管理模式需要以合同约定为基础，对项目设计、采购、施工、试运行等环节的风险管理工作进行落实，全方位管控EPC项目的工程质量、工程安全、工程造价等要素。结合设计牵头EPC 项目全过程风险的主要成因及其识别方法，可以从自然与社会风险、相关方管理风险、项目技术风险、项目商务风险、项目投标风险等角度入手采取相应的风险管理对策和风险防控措施。

# 1设计牵头 EPC 项目全过程风险的主要成因

# 1.1风险理论研究与实际应用存在差异

我国EPC项目全过程风险管理主要集中于施工方，很少涉及项目牵头的总承包项目风险管理。设计牵头EPC项目全过程风险管理指针对某一环节产生的风险采取相应的风险防范措施，但在实践过程中不仅缺少系统性、战略性的风险管理及应对措施，也并未针对动态化的风险管理过程制定相应的技术手段和方法[]。与此同时，设计牵头EPC项目全过程风险管理极为依赖专家经验，在实践过程中缺少系统性的风险识别和风险管理对策。

# 1.2合同管理难度大

在EPC项目实施过程中，项目业主只需要委托总承包商完成相关工作，在实践过程中项目业主很少参与项目组织、协调的过程，但这同样对总承包商的技术能力和综合管理能力提出了较高的要求，总承包商需要对内部及分包商之间的协调关系进行调整，这也在一定程度上加大了设计牵头EPC项目合同管理的难度[2²]。与此同时，项目设计方在工程设计环节，通常需要兼顾项目总体规划及工程施工管理相关工作，尤其在EPC项目实施的前期阶段，还需要进行项目招投标活动，确保EPC项目在激烈的市场竞争中占据一定的优势，并为设计牵头EPC项目的顺利实施奠定良好基础。但在初期阶段发包方尚未明确制定工作准则，承发包制定的合同条款也存在一定的纰漏，其与施工联合体方还可能产生各种合同纠纷，从而导致设计牵头EPC项目合同管理难度不断增大。

# 1.3EPC项目风险复杂

设计牵头EPC项目全过程风险具有复杂多样的特点，如水处理类EPC项目伴有较高的设计风险，在项目实施过程中需要满足高难度的工艺要求；大型复杂的EPC项目主要集中于施工风险，其对现场施工安全提出了较高的要求；交通类基础设施EPC项目则伴随着较大的社会风险，因此，在EPC项目风险识别、管控过程中需要采取不同的方法和手段。设计牵头EPC项目全过程不仅涉及复杂的接口及组织结构，而且在整个项目实施过程中伴随着各种不稳定的风险因素，使得EPC项目全过程风险发生的概率及其影响程度的不确定性增加。总承包商在EPC项目全过程风险管理工作中需要覆盖项目设计、采购、施工、试运行等多个环节，但在EPC项目实施的全生命周期内，每一阶段发生的风险具有较强的关联性和耦合性，由此导致设计牵头EPC项目全过程风险的种类复杂多样[3]。

# 1.4风险管理薄弱

设计牵头EPC项目在实施过程中，主要由设计方突破设计与施工之间的界面障碍，这也是加强设计方与施工方互动交流的重要切入点。在EPC项目实施的设计阶段，可以邀请施工专家参加边设计、边施工的过程，通过集中专家经验大幅度提高工程施工效率。在设计牵头下管控工程投资，还有助于减少承包商成本、提高项目施工效益。但大部分设计单位更注重轻资产运营的方式，在实际的项目管理工作中不具备承担大型财务风险的能力，使得设计单位在项目管理过程中被迫承担连带责任。与此同时，设计单位现有的复合型管理人才也相对较少，无法为EPC项目的外部沟通协调工作提供人力方面的支持。此外，设计牵头EPC项目全过程风险管理还缺少完善成熟的组织架构和制度体系，在实践过程中很难第一时间采取科学有效的应急响应措施，由此导致设计牵头EPC项目风险管理的基础较为薄弱。

# 2设计牵头 EPC项目全过程风险的主要识别方法

设计牵头EPC项目全过程风险管理主要涉及案例库法、流程分析法、合规义务识别法、溯源分析法、专家评估法等多种风险识别方法，根据EPC项目实施过程中可能发生的风险，设计单位可以结合实际的经营管理需求和项目管理要求构建一个系统的风险识别框架（见表1）。案例库法指采集、分析行业已有的案例记录，对EPC项目风险案例进行集中处理与整合，这种方法具有较强的启发性和广泛性，但很难针对性地处理某一特定的项目风险[4]。流程分析法指梳理总结EPC项目全过程风险管理的主要流程，逐一排查其中存在的风险。这种方法可以帮助企业解决一部分管理方面的问题，但其对项目管理流程分析人员的专业素养和技术能力提出了较高的要求。合规义务识别法指归纳、总结行业法律规范及相关管理规定，通过对比分析的方式识别项目风险。这种方法可以帮助企业明确合规经营的界限，但在实施过程中只能应付基础、宏观等层面的工作内容，不适用于复杂的EPC项目风险管理工作。溯源分析法指追溯项目问题发生的原因，通过这种方式挖掘项目可能发生的风险。利用该方法可以把握风险发生的源头，从现象看出问题的本质，但这种方法主要用于事后分析环节，很难第一时间生成相应的风险防控措施。专家评估法指参加行业内专家的深度访谈、座谈会等，通过这种方式客观评估、判断已发生的项目风险。该方法可以凭借专家经验对项目风险进行评估与完善，但这种方法较为依赖专家的主观经验和专业能力。

# 3设计牵头EPC项目全过程风险管理及应对措施

某EPC项目涉及总承包商、设计院、工程勘察院等多个承包单位，在项目实施过程中考虑到建设单位的专业能力有所欠缺，现邀请相关专业机构负责设计、BIM、造价咨询等工作。总承包商在项目设计、采购、施工等环节融合过程中，尝试采用钢结构、机电、精装修、智能化等专业分包同步进场的方式。在设计牵头EPC项目全过程风险管理过程中应初步采集风险事件相关信息，重点分析已建类似工程的数据资料、拟建项目数据资料、工程设计施工文件等基础信息，客观评估EPC项目实施过程中可能发生的问题事件和风险，在设计牵头EPC项目全过程风险管理带动下制定相应的初始风险因素清单，明确标出可能影响项目顺利实施及项目经济效益的各种风险因素，从而针对设计牵头EPC全过程风险管理的表现形式，采取相应的风险管理对策和风险防控措施。

# 3.1自然与社会风险

面对设计牵头EPC项目全过程风险管理中的自然灾害风险，在项目实施的初期阶段应积极采取防灾减灾措施，采用保险等方式完成经济补偿。在合同签订环节，相关分析人员应充分考虑自然灾害风险可能引起的不良后果，并积极向业主汇报相关事宜，同时保留相关的依据，以便于为项目业主补偿相关费用。若设计牵头EPC项目的建设条件有限，在标前答疑阶段应充分考虑项目资料缺失的可能，在投标报价环节预估这类风险费用。在项目投标活动中必须做好实地调研工作，掌握社会公众对EPC项目实施的看法和态度。在EPC项目实施的前期阶段还需要考虑手续滞后可能带来的影响，并将这类因素纳入合同签订的范畴，同时制定相应的争议谈判机制，从而有效避免后续可能发生的合同纠纷。在此基础上还需要科学规划水电热、人防等外部条件，同时预留出一定的费用资金[5]。此外，在设计牵头EPC项目全过程风险管理过程中应密切关注国家政策变化，对EPC项目风险管理对策做出相应的调整。一旦发现设计、施工规范等发生重大政策性变化，应第一时间保留相关数据资料，并同时将其反馈给业主方。

# 3.2相关方管理风险

在设计牵头EPC 项目全过程风险管理中应充分考虑发包方管理风险，采用书面形式、电子传输等多种方式传达重要的合同通知，明确合同双方的信息传达地址及联系方式，以便于精准落实业主方下达的口头指令。由于建设方和使用方涉及多个单位组成，因此，在EPC项目实施前应积极构建各方沟通交流渠道，采用联席会议纪要的方式采纳各方提出的建议。由于设计牵头EPC项目全过程的建设周期相对较长，一旦发现业主方领导变更，应立即指派专门的工作人员做好拜访汇报工作，从而有效应对业主方可能发生的需求变化。在联合体方管理风险防控过程中，应从源头上考察联合体成员方的人选，优先选择技术手段高、管理能力强、信誉良好的联合体成员方。在此基础上应明确规范遴选单位、遴选专家、遴选打分、遴选结果等评判指标，保证联合体成员方遴选程序的规范性[6]。与此同时，在联合体方管理风险防控过程中，还需要明确各环节的风险管理权责，如设计主导方享有的项目管控权限、项目管理的权责划分、设计优化产生效益的分配方式等。必要时还可以构建相应的合作评价机制，通过规范的评价机制与联合体方建立友好的合作关系。在分包方管理风险防控过程中，必须对分包商的资质和能力进行严格审查，细化分包合同履约中涉及的支付、变更等条款。另外，相关工作人员还需要严格审查分包单位的资信评价情况，客观评估分包单位的安全管理能力。在分包合同中必须明确规定工程材料品质、分包劳动资源投入、分包工程质量等指标。某些分包工程中含有的特殊工艺很可能对工程建设品质及建筑设计的艺术性产生重大影响，因此，在正式施工前必须严格审查分包工程技术人员的专业能力，组织相关技术人员参加相应的技术培训活动，以此为分包工程的建设品质提供人力方面的保障。

# 3.3项目技术风险

对于设计牵头EPC项目全过程风险管理中的项目技术风险，在项目实施的前期阶段必须严格遵守技术资料审批的程序，由负责人审查无误后再签字。在技术成果检查和验收环节，同样需要遵循特定的审核程序，待检查无误后再投入使用。在项目设计环节，必须全面排查工程图纸设计，并做好设计交底和施工图会审工作。在项目施工过程中应明确审查施工组织技术、生产加工技术等管控要点，科学编制分包招标的技术规格书，确保分包方明确项目规定的技术责任、技术需求等。在技术错漏风险防控过程中，应基于宏观角度审查项目设计图纸，规范各方交付的每一环节，充分考虑设计方与施工方在BIM模型建设方面体现出的差异，借助先进的软件技术构建设计、施工一体化的数字化平台。在技术工艺风险防控过程中，应优先选择熟练掌握特殊施工工艺的操作人员，并邀请其深入施工现场提供相应的技术指导，以免偏离于工程设计思想。在工程试运行前，应邀请行业专家制定相应的应急预案，并在施工现场提供科学的技术指导和技术协调服务，以免因工程试运行偏差而引发技术工艺风险。

# 3.4项目商务风险

对于设计牵头EPC项目全过程风险管理中的合同管理风险，在项目实施过程中应按照合同目标明确细化后续的工作内容，确保各方明确自身应履行的职责、合同条款、付款履约结算等工作内容，并将这些工作内容落实到具体的工作岗位及工作人员。在此基础上还需要明确规划各分包合同之间的界限，重点关注交叉项目中涉及的权利划分，避免分包合同在后续实施过程中产生不必要的纠纷。对于合同管理中涉及的书面资料，必须第一时间对其进行签认、存档，并保证附注文件的完整性。在成本管理风险防控过程中，应针对具有纵向受力作用的钢筋连接方式，项目监理工程师应全数检查，对于机械连接或者焊接接头的钢筋需要现场取样，委托具有检测资质的第三方检测机构进行检验;钢筋的安装要点在于核验各类钢筋的品种、数量和保护层厚度是否满足设计规范要求，当钢筋位置出现偏差时，应及时进行纠正，避免影响构筑物结构的稳定性，充分发挥钢筋在优化构筑物结构性能中的实际作用。混凝土是建筑工程施工中重要的结构材料，在项目施工过程中，项目监理必须采取一系列措施严格控制混凝土施工技术的每个环节，依据工程环境需要设计混凝土配合比，严格控制混凝土搅拌制备过程中的质量问题，在混凝土施工过程中控制浇筑振捣和养护成型的施工方法，通过各种措施保证混凝土施工技术质量满足设计规范要求，将其应用的控制工作落实到位，使混凝土施工技术在项目施工中发挥应有的作用，保证结构施工的安全质量。

# 3结束语

国家提倡“双碳”目标对建筑工程行业的高质量发展提出了新的挑战，加速了建筑业转型升级，在当前的时代背景下，为了实现建筑工程高质量、规范化和科学性的发展，项目监理人员需要更加注重施工现场质量的控制与监督。本文从项目监理工作程序、质量控制体系、监理人员素质和主要建筑材料等方面深入探究分析了建筑工程施工质量控制的策略与方法，进一步完善质量控制体系，强化监理人员的责任意识，落实工程项目全过程监理理念，加强建筑工程全过程的质量监督管理，实现建筑工程的高质量、高效益和高安全性的建设目标。

设计牵头EPC项目全过程风险管理的特点，统一制定预算计划及相应的成本管控指标，综合考虑项目实施过程中可能发生的风险，并留出一定的预算空间。其中成本管控流程主要涉及成本预测、成本核算、成本评估等多个环节，只有动态化掌控成本管控的全过程，才能第一时间采取具有针对性的成本管理风险防控措施。对于合同中涉及的付款和结算条款，必须严格遵守特定的结算原则，及时采集、归档所需的数据资料。在财税处理风险管控过程中，由于业主方需要分别向设计方和施工方付款，因此，需要明确设计优化后联合体方之间的效益分配原则，必要时可以建立相应的合作协商机制。当业主方向设计牵头方支付相应的款项，再由设计方向施工方支付相应的款项时，应严格审查收入确认、发票开具的合规性，在当地税务部门的配合下解决发票和税务问题。

# 3.5项目投标风险

在设计牵头EPC项目全过程风险管理中可以成立专门的投标工作小组，尽可能固定投标小组的成员，以免引起相应的投标组织风险。在重大投标项目实施过程中应定期组织各小组参加相应的沟通交流会议，以此保证对接的流畅性。与联合体成员方建立合作关系时，还需要不断完善联合体成员方之间的投标决策机制以及领导汇报机制，以免重复不必要的汇报流程。与此同时，在投标报价风险管控过程中,应优先确定总承包商的工作范畴，有针对性地调整投标报价，并根据业主招标需求编制相应的技术方案，以免出现报价漏项的问题。在重大项目开展前应积极做好内部评审工作，客观评估客户信用信息及其偿还能力，以实地考察的方式确定当地经济发展情况和地方政策，从而提高自身具有的竞争实力和竞争条件。

# 4结束语

在设计牵头EPC项目实施过程中必须严格遵守相应的法律规范及行业准则，在EPC项目风险管理过程中做好风险分析、风险识别、风险评估、风险预警、风险应对等工作，通过加强事前风险防控、事中风险管理、事后风险应对等流程，帮助企业降低EPC项目全过程风险发生的概率，真正为企业健康长远发展提供良好保障。
