
# 第4 章 LN送变电公司电网运维风险识别

# 4.1 电网运维风险因素分析

# 4.1.1 技术风险

技术风险是指在电网运维过程中，由于技术缺陷、技术发展滞后等因素引发的风险。当前，LN 送变电公司在输电线路带电作业、变电站智能巡检等方面已积极应用新技术，但仍面临诸多技术风险因素。

一是设备技术更新换代加快，相关技术标准、规程跟进不及时，个别新型设备运维“无章可循”。二是技改项目论证不充分，部分投入使用的新技术可靠性、实用性不高。如某变电站智能机器人项目前期可行性论证不足，导致投运初期频繁出现故障告警，影响了现场作业效率。三是关键核心技术受制于人，个别进口设备维修维护难度加大。四是部分自主研发项目难以完全适应现场需求，实用化程度不高。如某换流站阀厅智能监测系统部分模块功能“用不上”，造成资源浪费。五是技术创新的引领作用发挥不充分，成果转化应用速度慢。输变电运维领域一些前沿探索性课题、试点示范项目因缺乏持续投入，进展缓慢。

# 4.1.2 管理风险

管理风险泛指因公司治理结构、管控模式、业务流程等方面存在不足，导致电网运维工作效率低下、决策失误的风险。LN送变电公司目前管理体制机制还不够完善，管理风险不容忽视。

一是“三重一大”决策机制不健全，重大事项决策的科学性、规范性有待加强。个别工程建设、设备采购、人事任免等事关公司发展的重大问题没有经过充分论证，决策片面性强。二是公司本部与基层单位权责划分不均衡，事权与财权错位。有些应该属地化管理的具体事项，公司层面管得过多、过细；有些应该集中统筹的关键资源，基层单位掌控过度。三是同质化管理问题仍比较突出，一些管理制度“上下一般粗”，缺乏必要弹性。对不同地区、不同业态的基层单位实行“一刀切”，基层反映“下沉不进去”。四是过程管控存在薄弱环节，埋下管理风险隐患。如个别招标采购环节把关不严，廉洁风险易发多发；部分外包队伍管理不到位，野蛮施工

时有发生。

# 4.1.3 环境风险

环境风险主要是指外部宏观环境、自然环境、社会舆论环境等因素变化，对电网运维工作产生不利影响。近年来，LN 送变电公司环境风险日益凸显，不容忽视。一是经济下行压力加大，工程业主资金链趋紧，工程款拖欠时有发生，对公司造成较大资金占用。二是自然灾害频发，台风、暴雨、冰雪等极端天气对电网设备威胁加剧。据统计，近5年，因自然灾害导致的电网运维费用年均增长 $20 \%$ 以上。三是输电线路沿线征地、迁改难度加大。随着环保、人居环境要求提高，部分线路工程受阻，运维不便。四是电网项目社会关注度提升，个别工程因与当地经济社会发展、居民生产生活关联度高，稍有风吹草动，极易引发负面舆情，从而造成电网运维困难。五是网络安全形势日趋严峻，电力系统面临的网络攻击风险加剧。据不完全统计，“十三五”期间，LN 送变电公司所辖电网监控系统遭遇网络入侵事件数10起。

# 4.1.4 人身风险

人身风险泛指电网运维人员在作业过程中，因工作环境恶劣、安全意识淡薄、违章指挥等因素引发的人身伤亡风险。输电线路检修运维、变电站设备巡视处理等工作，作业环境复杂多变，人身风险因素较多。一是作业环境存在诸多不确定性因素。露天作业易受雷电、大风等灾害性天气影响；山区、林区作业地形复杂、进出不便；夜间作业、带电作业人身风险高。二是个人防护意识淡薄。作业前安全交底流于形式，个别员工违章操作、不按规程作业时有发生。三是管理人员安全意识不强。个别带班领导安全责任意识不到位，现场履职尽责不够。四是应急处置能力不足。个别基层单位抢险救援、伤员救治预案针对性不强，一旦发生人身伤亡事故，很可能错失最佳处置时机。五是培训教育针对性不强。安全教育“红线意识”树立不牢，分层分类开展“四不两直”等实操性培训不够。

# 4.1.5 设备风险

设备风险是指因一次设备、二次设备及附属设施存在质量缺陷或自然老化，影响电网安全稳定运行的风险。高压电气设备作为电网的核心组成部分，其健康状况直接关系到整个电力系统的可靠性。当前，LN送变电公司设备风险形势不容乐观。一是设备质量参差不齐。个别批次变压器、断路器等关键设备存在出厂缺陷，检修中隐患不断。二是设备检修不到位。个别地区受地形、天气等因素影响，导致输电线路、变电站等电网“健康体检”不彻底。三是 “带病运行”现象时有发生。个别基层运维保线站为确保可靠供电，存在“能用就用”“出了问题再说”的侥幸心理，对设备缺陷整治“睁一只眼闭一只眼”。四是备品备件管理薄弱。关键备品备件储备不足，个别设备出现故障后，因备品备件短缺影响抢修进度。五是设备风险排查不够精准。设备缺陷信息归集不全面，设备状态评价不够科学，风险预警不及时。

# 4.2 电网运维的风险识别

# 4.2.1 风险识别的方法选择

电网运维风险识别是风险管理的首要环节，识别的全面性和准确性直接影响后续风险评估和应对的有效性。LN 送变电公司电网运维业务涉及环节多、专业性强，各类风险错综复杂、相互交织，全面识别风险不是一件容易的事。

根据GB/T24353-2009《项目群风险管理指南》，风险识别的常用方法包括德尔菲法、头脑风暴法、故障树分析法、流程图分析法、SWOT分析法等。

在选择风险识别方法时，综合考虑了LN送变电公司的实际情况和各种方法的特点。德尔菲法虽然能够汇集专家意见，但耗时较长，难以快速响应公司动态变化的风险环境。故障树分析法主要适用于系统可靠性分析，难以全面覆盖电网运维中的管理、环境等非技术性风险。SWOT分析法更侧重于战略层面的分析，不够聚焦于具体的运维风险识别。相比之下，头脑风暴法能够在短时间内集思广益，快速识别各类潜在风险；流程图分析法则可以系统性地梳理电网运维各环节，查找薄弱点。这两种方法结合使用，既能保证风险识别的全面性和深入性，又能适应公司快速决策的需求，因此更适合 LN 送变电公司的实际情况。考虑到LN送变电公司实际情况，拟采用头脑风暴法和流程图分析法相结合的方式开展风险识别。

头脑风暴法是在轻松、愉悦、民主的环境中，通过群体成员集思广益，激发灵感，产生创意的一种讨论方式。它鼓励成员畅所欲言，在头脑风暴过程中，任何人都可以随意提出各种想法和观点，所有观点均被记录下来，暂不加以评论、质疑和批判，待头脑风暴结束后再对这些观点进行归纳、整理、评估和修正。

流程图分析法是对业务流程的每一个环节逐一进行调查分析，从中发现潜在的风险因素的方法。该方法通过绘制业务流程图，形象直观地展现业务活动的衔接过程，有助于查找流程中易发生风险的薄弱环节。流程图分析法的风险识别质量在很大程度上取决于绘制流程图的详细程度以及分析人员对业务流程的熟悉程度。

考虑到LN送变电公司下辖多个专业、跨越不同区域，采用头脑风暴法可以集中各方人员的集体智慧，快速梳理出“点多、面广、线长”的风险因素；而采用流程图分析法，可以“解剖麻雀”，细致分析每一业务环节可能存在的风险隐患，查缺补漏。同时，流程图分析法通过可视化的方式直观展现风险点，也有利于风险信息在公司上下的传递共享。

# 4.2.2 风险识别的步骤流程

根据头脑风暴法和流程图分析法的基本原理和实施要求，LN送变电公司电网运维风险识别的基本步骤如下：

（1）组建风险识别工作小组

为了系统、全面地开展风险识别工作，本研究组建了一个专业的风险识别工作小组。小组由 12 名成员组成，包括 4 名来自公司运维部门的资深工程师，2名安全管理专员，3 名基层运维人员，以及 3 名外部电力行业风险管理专家。这种配置既保证了专业性，又兼顾了实践经验。小组成员分别负责输电、变电、配电、调控等不同专业领域的风险识别工作，确保全面覆盖电网运维的各个方面。同时，小组还通过邀请其他相关部门人员参与讨论的方式，进一步扩大风险识别的覆盖面和深度。这种专业化的工作小组结构，既符合论文研究的实际需求，又能确保风险识别工作的质量和全面性。

（2）开展头脑风暴会议。

邀请公司内部员工和外部专家组成专家组，利用头脑风暴法识别风险因素。专家组成员应具备丰富的电网运维工作经验，专业领域覆盖全面。专家组名单见表4.1。

# 4.2.3 风险识别清单

在组建风险识别工作小组后，按照既定步骤开展风险识别工作。首先，工作小组召开初步识别会议，采用头脑风暴法，鼓励成员自由发表意见，广泛收集可能存在的风险因素。在这个阶段，工作小组共识别出35项潜在风险，并将其归纳为技术风险、管理风险、环境风险、人身风险和设备风险五大类。随后，小组进行了第二轮讨论，对每一项识别出的风险进行深入分析。在这个过程中，成员们就风险的合并、新增、定义和分类等问题展开了热烈讨论。例如，有成员提议将“舆情风险”纳入环境风险类别，考虑到公众对电力设施建设的关注度日益提高；对于“技术储备不足风险”的分类，经过讨论最终达成共识，将其归为技术风险，但在描述中强调了管理因素的影响。

经过两轮讨论和修改，工作小组对风险清单进行了优化和调整。一些影响较小或可能性极低的风险被剔除，新发现的重要风险被补充进来。最终，工作小组将风险数量精简至20项，形成了一份相对全面而精炼的风险识别清单。为验证清单的合理性和全面性，小组邀请了公司其他部门的代表和外部专家进行评审。评审专家对清单给予了积极评价，同时也提出了一些建设性意见。根据这些反馈，工作小组对清单进行了最后的微调和完善。

最终形成的风险识别清单不仅涵盖了LN送变电公司电网运维中可能面临的主要风险，还反映了当前电力行业的新趋势和新挑战。例如，清单中包含了信息化管理风险，反映了电网向智能化、数字化转型过程中可能面临的新型风险。这份经过多轮讨论和优化的风险识别清单，为后续的风险评估和管理工作奠定了坚实的基础。通过系统化、科学化的识别过程，确保了风险清单的全面性、准确性和实用性，为LN 送变电公司的电网运维风险管理提供了可靠的基础数据。

# 第5 章 LN送变电公司电网运维风险评估

# 5.1 风险评估方法选取

风险评估方法众多，常用的包括定性分析法、定量分析法和定性定量相结合的方法。定性分析法主要有德尔菲法、专家评估法等；定量分析法包括概率分析法、蒙特卡洛模拟法等；定性定量相结合的方法有层次分析法、模糊综合评价法、灰色系统理论等。考虑到LN送变电公司电网运维风险的复杂性和多样性，单一的定性或定量方法难以全面准确地评估风险。因此，选择了层次分析法和模糊综合评价法相结合的方式。层次分析法能够将复杂问题系统化、层次化，有效处理定性与定量指标，适合构建电网运维风险指标体系并确定权重；模糊综合评价法则可以很好地处理风险评估中的模糊性和不确定性，能够综合多种因素对风险进行全面评判。这两种方法的结合既能保证评估的科学性和系统性，又能充分考虑电网运维风险的模糊特征，因此更适合LN送变电公司的实际情况。

# 5.1.1 层次分析法

层次分析法（Analytic Hierarchy Process,AHP）是美国运筹学家 T.L.Saaty 于 20世纪70年代提出的一种将定性问题定量化的系统分析方法。它通过将复杂问题分解为若干层次和若干因素，进行两两比较和计算，得出各因素的权重，为决策提供依据。

层次分析法的基本步骤如下：

（1）建立层次结构模型。将决策问题分解为若干层次，每个层次包括若干个要素，要素之间存在隶属关系。一般分为目标层、准则层和方案层。

（2）构造判断矩阵。采用1-9标度对同层次因素两两比较，确定相对重要性，构造成对比较矩阵，即判断矩阵A:

$$
\begin{array} { r } { { 1 } } { { \mathsf { a } } _ { 1 2 } \quad { \cdots } \quad { \mathsf { a } } _ { \mathrm { 1 n } } } \\ { { \mathsf { A } } = ( { \mathsf { a } } _ { \mathrm { i j } } ) _ { \mathrm { n } \times \mathrm { n } } = ( { \mathsf { a } } _ { 2 1 } ^ { \mathsf { a } _ { 2 1 } } { \mathsf { \Omega } } _ { \vdots } ^ { 1 } \quad { \cdots } \quad { \mathsf { a } } _ { \mathrm { 2 n } } ) } \\ { { \mathsf { a } } _ { \mathrm { n } 1 } \quad { \mathsf { a } } _ { \mathrm { n } 2 } \quad { \cdots } \quad { \mathsf { \Omega } } _ { 1 } ^ { } } \end{array}
$$

其中， ${ \tt a } _ { \mathrm { i j } }$ 表示元素 i 对元素 $\mathrm { j }$ 的相对重要性， $\mathrm { a _ { i j } > 0 , a _ { i j } = 1 / a _ { j i } , a _ { i i } = 1 , i , j = }$ 1,2,⋯,n。

（3）层次单排序。对判断矩阵进行归一化处理，计算判断矩阵的最大特征值λ及其对应的特征向量W，对W进行归一化，得到该层次对上一层次某元素的相对重要性权重向量。

$$
\mathsf { A W } = \lambda _ { \operatorname* { m a x } } \mathsf { W }
$$

（4）一致性检验。由于判断矩阵是人的主观判断结果，可能存在不一致现象。为保证结果可信，需进行一致性检验。定义一致性指标CI:

$$
\begin{array} { r } { \mathsf { C I } = \frac { \lambda _ { \operatorname* { m a x } } - \mathrm { n } } { \mathrm { n } - 1 } } \end{array}
$$

CI 越接近于 0，判断矩阵的一致性越好。再查找平均随机一致性指标RI，计算一致性比例CR:

$$
\begin{array} { r } { \mathrm { C R } = \frac { \mathrm { C I } } { \mathrm { R I } } } \end{array}
$$

当 $\mathrm { C R } { < } 0 . 1$ 时，认为判断矩阵通过一致性检验，权重向量可以使用；否则需要修正判断矩阵，直至通过检验。

（5）层次总排序。如果决策问题有多个层次，则需要自下而上逐层计算，得到最底层方案对于最高层目标的组合权重，即层次总排序。设上一层次A包含 $\mathbf { m }$ 个因素 $\mathrm { A } _ { 1 } , \mathrm { A } _ { 2 } , \cdots , \mathrm { A } _ { \mathrm { m } }$ ，其权重向量为 $\mathbf { a } = ( \mathbf { a } _ { 1 } , \mathbf { a } _ { 2 } , \cdots , \mathbf { a } _ { \mathrm { m } } ) ^ { \mathrm { T } }$ ，本层 B 包含 $\mathbf { n }$ 个因素$\mathsf { B } _ { 1 } , \mathsf { B } _ { 2 } , \cdots , \mathsf { B } _ { \mathrm { n } }$ ，则B层的总排序权重计算公式为：

$$
\begin{array} { r } { \mathsf { W } _ { \mathrm { i } } ^ { \mathrm { B } } = \sum _ { \mathrm { j } = 1 } ^ { \mathrm { m } } \mathsf { a } _ { \mathrm { j } } \mathsf { W } _ { \mathrm { i j } } ^ { \mathrm { B } } , \mathrm { i } = 1 , 2 , \cdots , \mathrm { n } } \end{array}
$$

# 5.1.2 模糊综合评价法

模糊综合评价法是运用模糊数学的原理和方法，对受多因素影响的事物进行综合评判的一种数学方法。它充分考虑了人的主观判断的模糊性，能够定量描述难以用精确数学语言描述的不确定性问题，在风险评估领域得到广泛应用。

模糊综合评价法的基本步骤如下：

（1）确定评价指标集 $\mathrm { U } = \{ \mathrm { u } _ { 1 } , \mathrm { u } _ { 2 } , \cdots , \mathrm { u } _ { \mathrm { n } } \}$ 和评语等级集 $\mathsf { V } = \{ \mathsf { v } _ { 1 } , \mathsf { v } _ { 2 } , \cdots , \mathsf { v } _ { \mathrm { m } } \} .$ 。U为评价对象的特性或影响因素， $\mathrm { V }$ 为对评价对象作出的判断，如：$\begin{array} { r } { \mathrm { { V } } = \left\{ \begin{array} { r l } \end{array} \right. } \end{array}$ {高风险，较高风险，中等风险，较低风险，低风险}。

（2）确定指标权重。可以采用层次分析法、专家打分法等方法，得到指标权重集 $\mathrm { A } = ( \mathrm { a } _ { 1 } , \mathrm { a } _ { 2 } , \cdots , \mathrm { a } _ { \mathrm { n } } )$ ，满足 $\mathrm { . } \sum _ { \mathrm { i } = 1 } ^ { \mathrm { n } } \mathsf { a } _ { \mathrm { i } } = 1 , \mathsf { a } _ { \mathrm { i } } \geq 0 , \mathrm { i } = 1 , 2 , \cdots , \mathsf { n } _ { \circ }$ 。

（3）确定隶属度矩阵。对评价对象从各评价指标 $\mathrm { u _ { i } }$ 出发进行单因素评价，得到隶属度矩阵R:

$$
\begin{array} { r } { { \bf r } _ { 1 1 } \quad { \bf r } _ { 1 2 } \quad \cdots \quad { \bf r } _ { 1 \mathrm { m } } } \\ { { \bf R } = ( { \bf r } _ { \mathrm { i j } } ) _ { \mathrm { n } \times \mathrm { m } } = ( { \bf r } _ { \mathrm { ~ \dot { ~ 2 } 1 } } ^ { \mathrm { r } _ { 2 1 } } \quad { \bf r } _ { 2 2 } \quad \cdots \quad { \bf r } _ { 2 \mathrm { m } } } \\ { \quad \vdots \quad \vdots \quad \ddots \quad \vdots } \\ { { \bf r } _ { \mathrm { n 1 } } \quad { \bf r } _ { \mathrm { n 2 } } \quad \cdots \quad { \bf r } _ { \mathrm { n m } } } \end{array}
$$

其中， $\mathrm { r _ { i j } }$ 表示从指标u看，评价对象对评语 $\mathbf { v } _ { \mathrm { j } }$ 的隶属度，可以通过专家打分等方法获得。

（4）进行模糊综合评判。采用适当的模糊算子，对指标权重集A和隶属度矩阵R进行合成运算，得到综合评判结果B:

$$
\mathrm { \sf { B } } = \mathrm { \sf { A } } \circ \mathrm { \sf { R } } = ( \mathrm { \bf { b } } _ { 1 } , \mathrm { \bf { b } } _ { 2 } , \cdots , \mathrm { \bf { b } } _ { \mathrm { \scriptscriptstyle { m } } } )
$$

其中， $\begin{array} { r } { \mathrm { b } _ { \mathrm { j } } = \operatorname* { m i n } \{ 1 , \sum _ { \mathrm { i = 1 } } ^ { \mathrm { n } } \mathrm { a } _ { \mathrm { i } } \cdot \mathrm { r } _ { \mathrm { i j } } \} , \mathrm { j } = 1 , 2 , \cdots , \mathrm { m } } \end{array}$ ，表示评价对象对评语等级vj的综合隶属度。B 反映了评价对象在各评语等级上的分布情况。

（5）评判结果分析。根据B的隶属度分布，可以得到评价对象的风险等级。常用方法有最大隶属度原则、加权平均原则等。如果 $\mathbf b _ { \mathbf k } = \operatorname* { m a x } \{ { \mathbf b } _ { 1 } , { \mathbf b } _ { 2 } , \cdots , { \mathbf b } _ { \mathrm { m } } \}$ ，则评价对象属于风险等级 $\mathbf { v _ { \mathrm { k } } }$ 。

在LN送变电公司电网运维风险评估中，需要将风险因素作为评价指标，将风险等级作为评语等级，邀请专家对各风险因素的重要性和隶属度进行打分，构造判断矩阵和隶属度矩阵，通过模糊合成运算得到综合风险评判结果。

# 5.2.1 风险评估指标选取的原则

科学合理地选取风险评估指标是正确开展风险评估的前提。LN送变电公司在构建电网运维风险评估指标体系时，应遵循以下原则：

（1）系统性原则。电网运维风险评估指标应覆盖风险管理的各个环节，涵盖风险识别、风险分析、风险评价和风险应对等全过程，构成完整的指标体系，全面反映电网运维风险的各种特征和影响因素。

（2）针对性原则。电网运维风险具有其特殊性，受自然环境、设备状态、人员技能等因素的综合影响。选取评估指标时，应立足电力行业实际，充分考虑影响电网安全稳定运行的关键因素，突出电网运维领域的特殊风险。

（3）可操作性原则。风险评估指标选取不能过于宏观和笼统，应具有明确的统计口径和计算方法，便于实际应用。要尽量选择定量指标，对于难以定量的因素，也要制定明确的定性判断标准，确保评估过程可操作、评估结果可比较。

（4）数据可获得性原则。设计评估指标时，要充分考虑企业内部积累的数据和外部可获得的数据，选择容易收集、易于计量的指标。对于暂时无法获得数据支撑的指标，可以通过合理假设和专家经验判断等方式进行补充。

（5）动态优化原则。电网运维风险具有动态演变的特点，风险因素和风险水平会随着内外部环境的变化而变化。评估指标体系的设计应具有一定的前瞻性和稳定性，同时要定期评估指标设置的合理性，必要时进行动态优化调整。

二是完善智能运维配套措施。加强顶层设计，制定智能运维管理办法，完善工作标准和技术规范；加大资金投入，支持开展智能传感、智能分析、智能决策等关键技术攻关，为智能运维发展注入新动能；优化业务流程，推动业财一体化管理，确保智能运维高效协同运转；建设智能运维大数据中心，推动各专业系统的数据汇聚共享；强化人才队伍建设，创新人才培养和激励机制，加快培养一支创新型、复合型的智能运维人才队伍；加强统筹协调，整合内外资源，积极与高校、科研院所、设备制造商展开产学研用合作。通过政策、资金、人才、数据、机制等要素协同发力，为智能运维体系建设提供强有力支撑。

# 6.2.5 强化应急管理，扩大应急力量

极端天气、外力破坏等突发事件已成为影响电网安全运行的重要因素，必须重视电力应急管理和应急力量建设，不断健全完善应对机制，着力提高应急处置能力。

一是健全应急管理体制机制。根据国家应急管理改革要求，加快推进公司应急管理体系和工作机制改革，进一步明晰决策指挥、风险监测、应急处置、保障协同等重点职责，形成统分结合、上下联动、反应灵敏、协同有序的应急指挥体系；系统梳理评估各类应急预案，着力提高针对性、可操作性，推动应急预案与综合防灾、安全生产、应急避险等各专项工作有效衔接；建立健全应急管理责任制，压实属地管理责任和部门监管责任，层层传导压力确保责任落实不留死角。

二是加强基层应急能力建设。加强应急管理队伍建设，合理确定人员编制，加大教育培训力度，着力提高应急指挥人员的综合素质；强化应急装备配备，结合辖区灾害特点和重点防范需求，加大抢修车辆、应急发电机组、无人机等装备购置更新力度，提高装备配备的针对性和实用性；健全基层应急物资保障体系，科学核定物资储备品类和规模，完善物资紧急调拨机制，确保关键时刻物资供应及时到位；深化区域应急协作，加强与地方政府及相关部门的应急联动，建立重大灾害预警、会商、联合处置等机制，形成监测预警、应急处置、恢复重建一体化的区域应急协同格局。
