
# 第 2 章 项目风险管理概述

# 2.1 项目风险管理的概念

（1）风险的概念与特征

“风险”一词，最早在远古时期以出海捕鱼为生的渔民中形成，渔民发现海上的风浪会给出海捕鱼带来不确定和损失，所以每次出海前都要祈求风平浪静，让自己满载而归。在渔民眼中“风”就会产生“险”，这也是“风险”的由来。风险是指“某种危险发生的可能性与后果的组合”，也有解释为“有可能发生的危险”，无论哪种解释，都离不开“可能性”和“危险”两个关键字。这也说明任何风险都有一定的概率会发生，并且发生必然会形成损失。风险通常有以下特征：普遍性、必然性、不确定性、损失性。

# （2）项目风险的概念

项目风险是指在项目的全生命周期内，不确定因素导致项目发展偏离预期，并产生损失。例如不断变化的外部环境：甲方需求、政策变化、市场波动等，以及公司内部的不确定因素：管理缺失、人员变动、资金流问题等。项目风险管理就是在风险发生前，通过有效手段对将要发生的风险进行识别，对风险会导致的损失进行预判，并制定有效的风险防控及应对措施。项目的风险管理过程一般包括以下几个步骤：即风险识别、风险评估、风险应对、风险监控[23]。

# 2.2 项目风险管理的过程

# 2.2.1 项目风险识别

项目的风险识别作为项目风险管理的第一步，是项目风险管理重要的一个环节。只有正确地识别出项目所面临的全部风险，才能有效地进行项目风险管理。运用头脑风暴法对项目进行初步风险识别，形成初始风险清单。结合WBS-RBS矩阵法在，初步识别结果的基础上进行再次确认，剔除与本项目无关的风险，整理得到风险清单。探寻了风险来源主要是从财务风险、运行风险、安全风险、人员风险、合规风险、舆论风险六个大类的 20 条风险因素，为后续的风险评估分析奠定了基础。

# 2.2.2 项目风险评估

在识别出项目存在的风险基础上，分别层次分析法、模糊综合评价法，对所

有风险进行评估。结合各项风险发生的概率以及风险可能导致的损失，对各项风险进行评估，并根据分析结果对各项风险进行排序，确保评估结果具有指导意义。

# 2.2.3 项目风险应对

通过有效的风险识别和评估，可以得到项目存在哪些风险、风险发生的可能性以及风险可能引起的损失，在针对发生概率较大、发生后引起损失较大的制定应对措施，降低风险发生的概率，减小风险发生后引起的损失。风险的应对策略有以下几种：风险回避、风险减轻、风险转移、风险接受。

# （1）风险回避

在应对危害程度较高的风险或不可接受的风险时，风险回避是最常用也最有效的应对策略。风险回避是通过改变可能会导致风险发生的工作计划，从而使风险发生的概率降为0，从源头避免风险的发生。这是一种在风险发生之前主动做出调整的风险应对策略，相比于其他三种应对策略，风险回避的应对效果是最彻底的。

# （2）风险减轻

风险减轻是通过降低风险的发生概率，或者降低风险发生后的影响程度，从而降低风险的整体威胁值的一种风险应对策略。在应对不能通过风险回避或回避成本较高的风险时，往往采用风险减轻的应对策略。与风险回避不同，风险减轻不能从根源上消除风险，只能降低风险威胁。风险减轻是在风险应对时最常用的策略。

# （3）风险转移

风险转移是在风险发生概率和风险影响程度不发生改变的情况下，风险转移给第三方的一种风险应对策略。与风险回避不同，风险转移并不会从根源杜绝风险的发生，只是将风险转移至有风险应对经验或风险承受能力的一方，从而规避自身风险的一种应对策略。比如在项目运行时，对某一专业领域的工作内容外包给具有专业经验的企业，从而充分发挥承包方的管理经验，转移管理风险。但值得注意的是，风险转移往往会伴随着成本上升。在运用风险转移策略时，要从多方面充分考虑。

# （4）风险接受

根据风险管理人员是否意识到风险，风险接受措施可分为计划内风险接受和计划外风险接受。前者发生在风险管理人员没有意识到项目风险的存在或低估风险的后果时，而这些风险在实际项目实施过程中应该避免；后者发生在风险管理人员经过合理分析和评估后主动转移与风险相关的潜在损失时[24]。大多数管理者倾向于选择风险转移策略，但并非所有风险都是可转移的，或者转移这些风险是不经济的。我们必须自己承担这些风险。此外，在某些情况下，保留部分风险也是合理的[25]。

# 2.2.4 项目风险监控

风险监控是除了风险识别、评估和应对外的一种常态化的风险管理方式，通过对项目运行情况进行监控，及时发现新的风险，必要时可对项目重新进行风险识别、评估，并完善风险应对措施，从而形成一套完整的风险管理闭环模式与管理反馈机制。

# 2.3 项目风险管理的方法.

# 2.3.1 项目风险识别方法

（1）头脑风暴法

头脑风暴法（又称脑力激荡法），是一种以会议形式在短时间内产生众多推断的团队合作思考方法。具体步骤可分为： $\textcircled{1}$ 确定议题：明确会议要讨论的内容以及要解决的问题。 $\textcircled{2}$ 会前准备：收集会议议题相关材料，便于参会人员了解议题相关背景。 $\textcircled{3}$ 确定人选：以 8 至12 人为宜，人数过多导致参会人员发言时间较少，人数较少则不利于激发思维。 $\textcircled{4}$ 明确分工：拟定会议主持人、记录员。 $\textcircled{5}$ 规定纪律：会议期间参会人员集思广益，各抒己见，严禁批评，不得私下讨论，不先评论，会议内容全部记录。 $\textcircled{6}$ 掌握时间：会议时间过短会让参会人员难以畅所欲言，时间过长又容易让参会人员产生疲劳，一般时间控制在 30 分钟至 1 小时以内。

# （2）WBS-RBS 矩阵法

WBS-RBS 矩阵法运用 WBS（即 Work Breakdown Structure）定义潜在风险源的层级结构，每降一级则定义一个更精确、详细风险源的方法为风险分解结构（Risk breakdown Structure）。将 WBS 与 RBS 结合在一起，形成 RBM （RiskBreakdownmatrix）。首先需要将项目按照其内在结构或实施过程逐层分解成不同的工作单元。每个工作单元要求相对独立、内容单一；将项目的潜在风险源按照不同类型逐一排列成树状结构；以 WBS 最底层的作业包和 RBS 最底层的风险因子分别作行和列构建矩阵；根据实际情况，按照RBS对应每个WBS 节点进行风险识别[26]。此种方法能判断出风险发生的阶段，有较强针对性，便于更好的进行识别和管控。适用性广，尤其适用于大型复杂、组织结构多项目。

# 2.3.2 项目风险评估方法

（1）层次分析法

层次分析法（简称AHP)，是指将与决策总是有关的元素分解成目标、准则、方案等层次，在此基础之上进行定性和定量分析的决策方法，应用网络系统理论和多目标综合评价方法，提出的一种层次权重决策分析方法。层次分析法是将决策问题按总目标、各层子目标、评价准则直至具体的备择方案的顺序分解为不同的层次结构，然后用求解判断矩阵特征向量的办法，求得每一层次的各元素对上一层次某元素的优先权重，最后再加权和的方法递阶归并各备择方案对总目标的最终权重[27]。

（2）模糊综合评价法

模糊综合评价法是一种用于处理模糊信息和进行决策的数学方法，它能够将主观的、模糊的和不确定的信息进行量化和分析，从而提供一个相对客观的评价结果。模糊综合评价法的具体步骤可分为： $\textcircled{1}$ 确定评价指标、 $\textcircled{2}$ 设定模糊语言变量、 $\textcircled{3}$ 建立模糊评价矩阵、 $\textcircled{4}$ 求解评价权重、 $\textcircled{5}$ 进行模糊综合评价、 $\textcircled{6}$ 解模糊化处理等。在使用时需要层次分析法得出的权重值设置评价指标和权重，以确保评价结果的准确性和可靠性。在层次分析法的基础上进行模糊综合评价，这种方式结合了层次分析法的结构化与定量分析能力，以及模糊综合评价法对模糊性与不确定性的处理能力。它能够更全面地考虑指标间的关系和相对重要性，并通过模糊评价进行量化分析。
