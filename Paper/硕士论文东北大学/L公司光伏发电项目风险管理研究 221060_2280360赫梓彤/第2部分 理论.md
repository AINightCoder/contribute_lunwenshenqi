
# 第 2 章 项目风险管理概述

# 2.1 项目风险概述

# 2.1.1 项目风险涵义

在《怀德曼项目管理词汇手册》中，美国项目管理大师马克思·怀德曼将项目风险定义为在项目实施过程中可能对项目目标产生负面影响的事件或情况。项目风险是一个复杂而多维的概念，指的是在项目的整个生命周期中，由于各种不确定因素可能带来的潜在影响。这些影响可能导致项目目标无法实现，或产生与预期不同的结果。项目风险的复杂性体现在它可能来源于内部因素，如团队成员的能力、项目管理的方法、技术选择，也可能来源于外部因素，如市场变化、法律法规的变动、自然环境等。不利因素，如资源短缺、技术难题、团队冲突、预算超支、时间延误等，需要通过风险管理策略进行识别、评估和缓解，以减少它们对项目成功的潜在负面影响。同时，项目风险也可能包括有利因素，这些因素可能会带来额外的价值或机会，如新技术的应用提高项目效率，市场环境的变化带来新的商机。因此，项目团队需要通过有效的风险管理策略来识别和利用这些机会，以最大化项目的利益，同时控制不确定性，提高项目成功的可能性。

# 2.1.2 项目风险的类别及特点

（1）不确定性

项目风险的核心特征之一是其固有的不确定性。这种不确定性不仅体现在风险事件是否会发生上，还涉及风险发生的具体时间、持续时长以及可能对项目造成的实际影响程度。由于项目环境的复杂多变和信息的有限性，项目管理者往往难以准确预测风险的所有细节，这使得风险管理和控制变得极具挑战性。不确定性要求项目团队保持高度的警觉和灵活性，以便在风险事件发生时能够迅速做出反应。

# （2）相对性

项目风险的相对性体现在风险对项目活动主体的影响是相对的。不同的项目主体（如项目发起人、承包商、供应商等）对同一风险的承受能力和应对策略可能截然不同。此外，风险的大小和重要性也取决于项目主体的投入大小、资源状况、风险偏好以及项目目标等因素。因此，在进行风险管理时，需要充分考虑项目主体的特点和需求，制定出符合实际情况的风险管理计划[11]。

（3）可变性

项目风险在项目进展过程中是动态变化的。随着项目发展与外部环境演变，潜在新风险持续出现，且既存风险的影响力可能出现变动。例如，技术难题可能随着技术的进步而得到解决，市场风险可能随着市场需求的变化而增加或减少。这就要求项目管理者持续监控项目风险，及时调整风险管理策略，以应对风险的变化。

# （4）阶段性

项目风险在不同的项目阶段具有不同的特点和重要性。在项目启动阶段，范围风险和技术风险可能较为突出，因为项目目标和实施方案尚未明确，技术难题也可能尚未得到解决。而在项目执行阶段，进度风险和成本风险可能更为关键，因为项目需要按照预定的计划和预算进行实施。因此，项目风险管理需要根据项目阶段的特点和重点，制定相应的风险管理策略和措施。

（5）相互关联性

项目中的风险往往不是孤立存在的，而是相互关联的。一个风险事件的发生可能引发其他风险事件的发生或影响，形成风险链或风险网络。例如，技术难题可能导致项目进度延误，进而引发成本超支和客户满意度下降等一系列问题。因此，项目管理者在识别和应对风险时，需要充分考虑风险之间的关联性，制定出综合的风险管理策略，以最大限度地降低风险对项目的影响。

# 2.1.3 项目风险影响因素

（1）外部环境因素

项目的外部环境是一个复杂多变的系统，其中政策与法规的频繁调整或突然变动，往往直接关联到项目的合规性、运营成本乃至生存空间。例如，环保政策的收紧可能要求项目增加环保投入，调整生产工艺，从而增加成本负担。市场条件的起伏，如消费者偏好的快速变化、竞争对手的激烈竞争以及行业趋势的不可预测性，都可能对项目的市场需求、销售价格和盈利能力产生重大影响。此外，自然环境的不可预测性，如地震、洪水、飓风等自然灾害，不仅可能直接破坏项目设施，还可能中断供应链，导致项目进度严重延误。

（2）项目内部因素

项目内部的技术难度、资源分配以及团队能力，是构成项目风险的核心要素。技术难度的高低直接关系到项目的实施周期和成功率，技术不成熟或过于复杂可能导致项目频繁遇到问题，甚至无法按时完成。资源的合理分配是项目顺利进行的基础，包括人力、物力、财力的充足供应和高效利用。资源分配不均或不足，不仅会导致项目进度受阻，还可能引发成本超支和团队士气低落。团队能力则是项目成功的关键，团队成员的专业技能、经验、沟通能力和协作精神，都直接影响到项目的执行效率和质量水平。

（3）管理因素

项目管理的水平对项目的风险管控效果具有决定性影响。项目规划的科学性和合理性，是确保项目目标清晰、路径明确、资源优化的基础。风险管理策略的有效实施，包括风险的识别、评估、应对和监控，能够显著降低潜在风险的影响，保障项目的顺利进行。高效的沟通与协调机制，则是确保项目各方紧密合作、共同应对挑战的重要保障，它能够促进信息的及时传递和问题的快速解决，减少误解和冲突。

# （4）经济因素

项目的经济状况是影响项目风险的重要因素之一。资金来源的稳定性决定了项目能否持续获得必要的资金支持，资金成本的高低则直接影响到项目的盈利能力和投资回报率。宏观经济条件的变动，涉及经济增长速度的减缓、通货膨胀的升级或是货币政策的调整等，都可能对项目的市场需求、成本结构和盈利能力产生深远影响。

# （5）社会与文化因素

社会稳定性和文化差异也是项目风险中不可忽视的重要方面。社会动荡和政治不稳定可能给项目实施带来安全隐患，如罢工、骚乱等事件可能中断项目进度，损害项目资产。文化差异则可能导致沟通障碍和价值观冲突，影响项目的团队协作和客户关系。例如，在不同的文化背景下，员工在工作方法、决策流程以及时间认知上可能存在明显的差异，需要项目管理者采取适当的措施进行调和和适应。

（6）合同与法律关系

合同条款的明确性和公平性，以及项目过程中可能遇到的法律纠纷，都是影响项目风险的重要法律因素。合同条款的不完备或表述不清，可能导致双方对合同具体内容产生理解差异，从而引发争端与纷争。知识产权侵权、合同纠纷等法律问题，不仅可能给项目带来经济损失，还可能损害项目的声誉和品牌形象。因此，项目管理者需要高度重视合同与法律关系的管理，确保合同条款的清晰明确和合法合规，及时应对和解决可能出现的法律纠纷。

# 2.2 项目风险管理涵义

项目风险管理是项目管理中一个至关重要的环节，它涉及从项目启动到项目结束的每一个阶段。这是一个全面而复杂的流程，要求项目团队在规划、执行、监控和收尾阶段中，对潜在的风险进行细致的识别、深入的分析、精确的评估、有效的应对和持续的监控。其核心目标是将风险控制在管理层可接受的范围内，以确保项目能够顺利进行并达成预定目标。项目风险管理不仅仅是一个识别和分析风险的过程，它还涉及到采取相应的措施来应对这些风险。

在这个过程中，致力于两个主要目标：一、最大化利用积极因素对项目风险管理流程的正面影响；二、最小化消极因素对流程产生的负面影响。项目风险管理流程主要包括以下方面：识别风险、量化风险以及应对风险的对策[17]。这些策略旨在减轻、转移、避免或接受风险，以确保项目能够在一个可接受的风险水平下顺利进行。通过这种综合性的管理方法，能够更好地控制不确定性，提高项目成功的可能性，同时给项目赋予最大的价值。

# 2.3 项目风险管理内容

# 2.3.1 项目风险识别

项目风险管理流程的第一步是项目风险识别，项目风险识别的重要程度不言而喻。在项目 风险识别的过程中，必须全面客观地对待每一个可能的风险因素，以确保对所有的不利因素进行充分覆盖和分析。在项目风险识别阶段，可通过多种策略与工具对潜在不利因素进行评估与区分，以确保对所以可能的风险进行充分覆盖和分析。

（1）项目文件审查：仔细审查项目文档，如项目章程、项目计划、需求文档、合同文件等，以找出潜在的风险源。（2）确定信息收集技术：使用各种信息收集技术,如头脑风暴、德尔菲技术、访谈、问卷调查等，从项目团队、专家、利益相关者及其他信息源获取风险数据。（3）列表检查法：参考历史项目的经验、行业标准和组织过往的风险清单等，使用检查表来识别可能的风险。（4）假设分析：讨论项目假设或基础条件的有效性，以及这些假设被证明不准确时可能产生的风险。（5）因果分析：使用例如因果图（鱼骨图）、5Why分析等工具，探讨可能导致风险事件的原因。（6）组织和记录：将所有已识别的风险组织并记录下来，通常使用风险登记册或风险数据库。（7）风险分类：按照风险的本质与起因，将其划分为不同类型，包括但不限于技术风险、商业风险及组织风险等。以便更有效地管理。

（8）优先级初步判定：对识别的风险进行初步的重要性评估，确定哪些风险需要优先关注。

（9）风险所有者指定：为每个风险分配责任人（风险所有者），以便于进行后续的风险分析和管理活动。

（10）沟通和共享：将识别到的风险与项目团队和利益相关者共享，确保他们了解可能对项目产生影响的风险。

（11）持续监测与适时更新：项目风险识别是一项持续性工作，在整个项目的发展周期中定期复核与更新。以确认新的风险并更新风险清单。

# 2.3.2 项目风险评估

项目风险管理过程的核心环节之一便是风险评估，这一步骤旨在系统性地分析与评价项目中所蕴含的不确定性及潜在风险因素。这个过程的目的是确定单个风险对项目目标的影响，并综合评价这些风险。项目风险评估的基础是对风险事件发生的可能性、可能的结果范围和危害程度、预期发生的时间以及风险因素产生的风险事件的发生频率进行估计。有些研究者将其总结为不确定损伤事态及概率和后果的集合[12]。在实际操作中，项目风险评估是一个迭代的过程，需要随着项目的进展不断迭代和完善。同时也需要注意的是项目中存在的各种风险并不一定会真正的发生，因为，项目风险评估是一个动态的过程，随着项目进展和环境变化，风险评估可能需要更新和调整。

# 2.3.3 项目风险应对

项目风险应对，作为项目风险管理最终的环节，扮演着至关重要的角色。在对项 目风险进行精准识别与深入评估之后，需要依据这些风险的量化结果，为可能出现的各种风险制定并执行相应的策略和计划。这一过程的核心目标，在于最大限度地防范可能损害项目目标的风险，确保项目收益与目的的圆满达成。在应对风险时，需要采用多种策略来应对及管理风险。根据沈建明在《项目风险管理》一书中的阐述，主要有四种关键的风险处理方式：风险规避、风险转移、风险保留以及风险缓释。这些方法旨在保护项目免受不确定性的影响，促进其成功实施。不论选择哪种策略，关键在于确保风险应对措施得到充分的沟通、记录和监控。

总而言之，项目风险应对是项目风险管理不可或缺的一环。通过精心策划并实施 适当的风险应对策略，能够有效降低风险对项目目标的负面影响，保护项目的利益和目标不受损害。只有拥有了高效的风险管理和应对措施，才能确保项目的顺利进行和成功交付。

# 2.4 项目风险管理方法

# 2.4.1项目风险识别方法

项目风险识别是项目管理过程中最重要的一步，它涉及使用不同的方法和技巧来发现和记录可能影响项目目标的不确定事件或条件。在实际过程中这些方法既可以单独使用又可以组合起来使用，以确保广泛而全面地识别项目风险。项目的负责人通过查询和识别已生成的风险列表，可以据此制定最合适的风险管理策略。

（1）经验判断法

经验判断法在风险识别中是一个重要且基本的方法。它主要利用个人或集体的知识和经验，来识别和分析可能出现的风险因素。这一方法强调依赖于企业各层次有关人员的经验和知识。这些人员通常对市场形势、项目特点等有着较为深入的理解，能够提供关于风险因素的重要信息。经验判断法作为一种风险识别的方法，强调了人员经验和直觉在风险管理中的重要性。它不仅适用于缺少充分历史数据的情境，还可以与其他方法结合，为风险识别提供丰富的视角和深度。

# （2）头脑风暴法

头脑风暴法是 A·F·奥斯本于 1939 年提出，并在 1953 年正式发表的一种创意产生技法。它通过以一种会议组织群体讨论形式，鼓励参与者在一个没有批评的环境中自由地提出想法，以此来激发创新和解决问题。有效进行头脑风暴法的关键在于充分的准备和合理的组织。在正式开始头脑风暴之前，组织者需要对问题进行深入分析，并收集相关资料。这一步骤能够确保参与者对问题有清晰的认识，从而更有针对性地提出想法。头脑风暴法作为一种促进创意思考的方法，不仅有助于激发个人的创造力，还能增强团队的协作和决策能力[13]。

（3）德尔菲法

德尔菲法也被称为专家调查法，是一种通过循环迭代获取专家意见以实现预测或决策的策略。起源自 20 世纪中期，由美国兰德公司的研究人员所开发的技术。最初用于军事策略和武器系统的评估。其实施步骤是明确要解决的问题或要达到的目标，依据问题的性质，挑选相关领域的专家，通过多轮的问卷或意见征集，收集专家们的意见。每轮结束后，对意见进行汇总和分析，再将结果反馈给专家，进行下一轮征询，直至达成共识或满足其他终止条件。德尔菲法作为一种高效集中专家智慧的工具，在解决复杂问题和制定关键决策中显示出了其独特的价值。

# 2.4.2项目风险评估方法

风险评估大致可分为三类：定量评估、定性评估以及结合定量和定性的综合评估。因为，项目生命周期的各个阶段所表现出的风险形式不一，所以，需要区分风险评估方法和侧重点，使其关联性和变量特征保持一致。项目风险评估形成一个持续循环的过程，通过针对特定项目的风险评估，能够构建起一条评估链。这条链条对于项目管理者、执行人员以及项目受益方均具有实际价值。它不仅有助于实现资源的高效分配，还对监控项目进度、提升绩效管理能力等方面具有关键意义。在项目风险评估中每一种评估方法均有其适用特点和优势，目前，项目风险评估实操中均运用组合的方式进行评估。

（1）层次分析法

美国匹兹堡大学的 Thomas·L·Satty（1970）教授开创性地提出了层析分析法（AHP）。这一决策分析工具以其独特的系统化思维，将错综复杂的多目标决策问题转化为一个条理分明的框架。通过将决策目标细化为多个具体指标，并进一步将这些指标分解成若干层次，AHP 方法巧妙地结合了定性与定量分析，对各项指标进行精确的权重分配，能够指引决策者找到最优的解决方案。层析分析法因其简洁实用、适应性强的特点，在众多领域得到了广泛应用[14]。无论是项目风险管理、企业战略规划，还是市场营销、供应链管理乃至人力资源管理，AHP 都展现出了其强大的决策辅助能力。然而，尽管 AHP 方法具有显著的优势，但它也不可避免地存在一些局限性，如操作过程相对复杂，以及在权重分配时可能受到主观判断的影响。尽管如此，层析分析法作为一种科学的决策工具，其价值和影响力不容忽视。它不仅为决策者提供了一种系统化的思考模式，还为复杂问题的解决开辟了一条明晰的路径。

# （2）模糊综合评价法

模糊综合评价法是一种基于模糊数学理论，用于解决多因素决策问题的综合评价方法。是美国学者 L·A·Zadeth（1965）引入模糊集合的概念，构建模糊行为与活动的模型，并非意味着舍弃数学的严谨性，而是在数学框架内探索并管理模糊现象，采用精确的数学工具进行分析与描述。在实际情况下，许多决策问题都涉及到多个因素的综合考虑，且这些因素往往具有不确定性或模糊性，使得决策过程变得复杂。模糊综合评价法被广泛应用于企业员工考核、项目评估、风险分析等多个领域，其特点在于能够将模糊的、难以量化的评价问题转化为定量分析，从而得到更为科学、合理的评价结果。模糊综合评价法具有独特的模糊数学理论基础，是处理复杂多因素决策问题的一种有效的工具。

# 2.4.3项目风险应对措施

项目风险应对措施是项目管理中的最终环节，在识别和评估潜在风险之后，制定相应的策略来减轻这些风险对项目目标的影响，并降低风险发生后潜在的损失。通常，佐飞（2017）所提的风险应对策略大致涵盖四类，分别是风险规避、风险转移、风险减缓 、风险自留[21]。详细区分如下：

（1）风险规避

当遭遇可能造成极其严重或完全无法容忍的风险时，风险回避策略往往被视为最为常用且极为有效的应对手段。这种策略的核心在于通过调整可能引发风险的相关计划或行动步骤，进而将风险发生的可能性直接降至零，从根本上杜绝风险的出现[15]。风险回避是一种预防性的风险管理方法，在风险成形之前就积极地进行策略调整和规划变更。与其他三种风险管理策略相比，风险回避能够提供最彻底的解决方案，确保风险被完全规避，从而保护项目或企业免受潜在的负面影响。风险规避并不是对所有风险的一劳永逸的解决方案，需要项目管理者在精确评估风险的基础上，制定合理的规避计划并付诸执行。

（2）风险转移

风险转移是一种策略，通过该方式，一方将其承担的风险责任有组织地转移到另一方身上，而在这一过程中，风险发生的可能性及其可能带来的影响保持不变。对于已识别的高风险元素，特别是那些可能对项目财务或进度有重大影响而团队又无法完全控制的风险时，应当考虑使用风险转移策略。尽管风险转移被认为是应用范围最广、最有效的风险管理手段之一，但使用此方法时也需注意一些条件和限制。例如，风险转移的成本、法律约束以及可能对企业经营带来的影响等。总的来说，风险转移是一种重要的风险管理工具，但需要根据具体情况谨慎选择和使用。

（3）风险缓解

指针对项目明确存在的风险，应实施适度的策略以控制损害，确保公司能承担相应的风险。若无法有效应对特定风险，只能独自承担时，通过采用适当的限制性措施，旨在减缓潜在风险的影响。在实际操作中，通过预先消除或感知可能导致风险的情形，可以有效降低风险事件的实际发生概率，亦或是实施策略以减少风险爆发时的潜在损失。具体手段包括但不限于采用合作体投标的方式、采纳更为简化的技术流程、选择更加熟悉的供应商等，这些均是行之有效的风险缓释举措。
